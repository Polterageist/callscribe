# Conventions — TDD и уровни тестирования

Этот документ фиксирует **практику TDD** и **уровни тестирования** в Callscribe.

## Базовый цикл (обязательно, когда тестируемо)

- **Red**: сначала добавь/измени тест, который падает и описывает ожидаемое поведение.
- **Green**: внеси минимальные изменения в код, чтобы тест стал зелёным.
- **Refactor**: упрости/почисти код без изменения поведения (тесты должны оставаться зелёными).

Если исправляется баг, начинай с **регрессионного теста**, который воспроизводит старое неверное поведение.

## Уровни тестирования (физическая раскладка)

Тесты разделяются по назначению:

- **`tests/unit/`**: быстрые изолированные тесты логики (контроллеры, чистые функции, правила, инварианты). Без реальных subprocess/GUI.
- **`tests/ui/`**: тесты адаптеров/UI wiring с подменами (например, построение меню `pystray` и enable/disable логика).
- **`tests/integration/`**: тесты “швов” системы (subprocess, несколько процессов, файловая система, запуск `python -m callscribe`).

Новые тесты добавляй в **самый низкий** подходящий уровень (unit > ui > integration).

## Кроссплатформенный замысел

- По возможности проектируем швы (например, single-instance lock/activation) через небольшие абстракции, чтобы код и тесты оставались переносимыми на другие desktop OS.
- Даже если CI сейчас гоняет тесты только на целевой платформе, **код тестов** стараемся писать platform-agnostic: ограничиваем **исполнение** через CI/markers, а не “вшитые” OS-only skip’ы в тест-модули.

## Про ручное тестирование

Ручная проверка полезна для UX и первичной отладки, но:

- **Не полагайся на ручное тестирование как на единственный метод**, если поведение можно надёжно зафиксировать авто-тестом.
- Для багов и регрессий — приоритет авто-тесту, который будет защищать от повторного появления проблемы.

## Практические рекомендации

- Держи тесты **детерминированными**: таймауты, корректный cleanup процессов, отсутствие зависимостей от “живого” UI.
- В интеграционных тестах предпочитай **тестовые хуки** (env/flags), чтобы запуск был ограниченным по времени и не зависал.
